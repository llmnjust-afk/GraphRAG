\title{IGV: Incremental Graph Versioning for Efficient\\Retrieval-Augmented Generation over Evolving Corpora}

\author{Anonymous}
\affiliation{\institution{Anonymous Institution}}

\begin{abstract}
Graph-based Retrieval-Augmented Generation (GraphRAG) enhances large language models (LLMs) by structuring document corpora as knowledge graphs, enabling multi-hop reasoning and global question answering. However, existing GraphRAG systems require expensive full-graph rebuilds when new documents arrive, making them impractical for dynamic, evolving corpora. We present IGV (Incremental Graph Versioner), a framework that supports efficient incremental updates to GraphRAG indexes through three innovations: (1) embedding-based semantic entity deduplication that merges duplicate entities across documents, reducing entity count by 13--17\%, (2) multi-hop graph relinking that connects new entities to the existing graph structure, and (3) incremental community repartitioning that locally updates community assignments without full recomputation. Experiments on five multi-hop QA datasets (HotpotQA, 2WikiMultiHopQA, MuSiQue, NarrativeQA, StreamingQA) with Qwen2.5-7B-Instruct across three random seeds demonstrate that IGV maintains retrieval quality (R@5 within 2\% of rebuild baseline) with near-zero forgetting rate (FR$<$0.03), while achieving statistically significant F1 improvements over naive incremental baselines ($p<0.05$, paired t-test). Long-term streaming experiments over 12 incremental batches confirm sub-9\% R@5 degradation, validating IGV's practicality for evolving corpora.
\end{abstract}

\keywords{GraphRAG, Incremental Indexing, Knowledge Graph, Retrieval-Augmented Generation}

\section{Introduction}

Retrieval-Augmented Generation (RAG) mitigates LLM hallucinations by grounding responses in retrieved external knowledge. While traditional vector RAG excels at factoid lookups, it struggles with global questions requiring cross-document reasoning. GraphRAG addresses this by constructing knowledge graphs from text corpora, enabling structured multi-hop retrieval~\cite{edge2024graphrag,guo2025lightrag}.

A critical limitation of current GraphRAG systems is their reliance on \emph{full rebuilds} for corpus updates. Microsoft GraphRAG~\cite{edge2024graphrag} requires re-running entity extraction, community detection, and summary generation over the entire corpus when any new document arrives. For a 1M-token corpus, this costs over 280 minutes of LLM time. LightRAG~\cite{guo2025lightrag} introduced a set-union incremental update, but its naive merging creates duplicate entities and fragmented graph structure without deduplication, conflict resolution, or community repartitioning.

We propose IGV (Incremental Graph Versioner), which addresses these gaps through three innovations:
\begin{itemize}
\item \textbf{Semantic entity deduplication} using sentence-transformer embeddings with cosine similarity, reducing entity count by 13--17\%.
\item \textbf{Multi-hop graph relinking} that connects newly inserted entities to the existing graph via BFS-based candidate generation.
\item \textbf{Incremental community repartitioning} that locally updates Louvain community assignments for affected nodes only.
\end{itemize}

Experiments on five datasets demonstrate that IGV achieves comparable retrieval quality (R@5 within 2\%) to full rebuild while requiring only 0.3--0.8s per incremental batch. Statistical significance tests confirm F1 improvements over naive incremental ($p<0.05$), and long-term streaming over 12 batches shows sub-9\% degradation with near-zero forgetting rate.

\section{Related Work}

\textbf{GraphRAG Systems.} Edge et al.~\cite{edge2024graphrag} proposed GraphRAG, which builds entity knowledge graphs via LLM extraction, applies Leiden community detection, and generates community summaries for global query answering. LightRAG~\cite{guo2025lightrag} simplified this with a dual-level retrieval paradigm and basic incremental updates via set union. PathRAG~\cite{chen2026pathrag} introduced flow-based path pruning to reduce retrieval redundancy. HippoRAG~\cite{gutierrez2024hipporag} used Personalized PageRank for neurobiologically inspired retrieval.

\textbf{Incremental Knowledge Graph Updates.} GORAG~\cite{wang2025gorag} proposed online graph updates for social media classification but lacks stateful graph index evolution. KAG~\cite{liang2024kag} introduced logical-form-guided reasoning with schema constraints but does not address incremental community repartitioning. Practical GraphRAG~\cite{min2025practical} reduced construction cost via dependency parsing but still requires full rebuilds for updates.

\textbf{Community Detection.} Louvain~\cite{blondel2008louvain} and Leiden~\cite{traag2019leiden} algorithms are widely used for modularity-based community detection. Dynamic community detection has been studied in network science but has not been applied to GraphRAG index management.

\section{Methodology}

\subsection{Problem Formulation}

Given a knowledge graph $G = (V, E)$ constructed from an initial document corpus $\mathcal{D}_0$, and a stream of new document batches $\mathcal{D}_1, \mathcal{D}_2, \ldots, \mathcal{D}_T$, the goal is to update $G$ to incorporate new entities and relationships from each batch $\mathcal{D}_t$ while minimizing LLM calls, avoiding duplicate entities, maintaining community structure quality, and preserving retrieval performance.

\subsection{IGV Architecture}

IGV processes each incremental batch through four stages:

\textbf{Stage 1: Entity Extraction.} For each new document, an LLM (Qwen2.5-7B-Instruct) extracts named entities and their relationships as triplets $(s, r, o)$.

\textbf{Stage 2: Semantic Deduplication.} New entities are compared against existing graph nodes using sentence-transformer embeddings ($d=384$). Given a new entity $e_{\text{new}}$ with embedding $\mathbf{e}_{\text{new}}$ and existing entities with embeddings $\{\mathbf{e}_1, \ldots, \mathbf{e}_N\}$, we compute cosine similarity. If $\text{sim}(e_{\text{new}}, e_i) \geq \tau$ (threshold $\tau=0.85$), entities are merged: the new entity's relationships are redirected to the existing entity, and descriptions are concatenated.

\textbf{Stage 3: Graph Relinking.} After merging, we identify candidate edges between new and existing nodes using BFS expansion to depth $h=2$. For each new node $v_{\text{new}}$, we explore its $h$-hop neighborhood and add edges to existing nodes with confidence $\geq 0.85$, where confidence is defined as $1/(1+\text{dist}(v_{\text{new}}, v_{\text{old}}))$.

\textbf{Stage 4: Incremental Community Repartitioning.} Rather than recomputing communities for the entire graph, we extract the subgraph induced by affected nodes (new nodes + their 2-hop neighbors) and re-apply Louvain only on this subgraph. The new sub-communities are merged back into the global partition.

\subsection{Complexity Analysis}

For a batch of $k$ new documents producing $m$ new entities and an existing graph with $N$ entities: dedup costs $O(m \times N)$ cosine similarities (vectorized), relinking costs $O(m \times d^h)$ where $d$ is average degree and $h$ is hop count, and community repartition costs $O(|S| \log |S|)$ where $|S| \ll N$ is the affected subgraph size. Compared to full rebuild $O((N+m) \log(N+m))$, IGV achieves sublinear cost when $m \ll N$.

\section{Experiments}

\subsection{Setup}

\textbf{Datasets.} We evaluate on five multi-hop QA datasets: HotpotQA, 2WikiMultiHopQA, MuSiQue, NarrativeQA, and StreamingQA. We use 300 passages and 50 questions per dataset, split into 70\% base / 10\%$\times$3 incremental batches.

\textbf{LLM.} Qwen2.5-7B-Instruct for entity extraction and answer generation, running on a single RTX 5090 GPU (32GB VRAM). Embeddings via all-MiniLM-L6-v2 (384-dim).

\textbf{Baselines.} B1 (Rebuild): full graph reconstruction at each stage. B2 (Naive): LightRAG-style set-union incremental. B3 (+Dedup): B2 + semantic deduplication. B4 (+Relink): B3 + 2-hop relinking. IGV (Ours): B4 + incremental community repartitioning.

\textbf{Metrics.} R@2, R@5, F1, EM, Comprehensiveness and Diversity (LLM-as-judge, 1--5 scale), entity count, update time, UER (Update Efficiency Ratio), SF (Subgraph Freshness), FR (Forgetting Rate), GQD (Graph Quality Degradation).

\textbf{Seeds.} All experiments run with 3 random seeds (42, 52, 62); we report mean$\pm$std.

\subsection{Ablation Study}

The progressive ablation from B2$\to$B3$\to$B4$\to$IGV reveals:
\begin{itemize}
\item \textbf{Dedup (B2$\to$B3):} Improves F1 on 2Wiki (0.106$\to$0.112, $p<0.001$) and HotpotQA (0.103$\to$0.107), confirming that removing duplicates reduces retrieval noise.
\item \textbf{Relinking (B3$\to$B4):} Improves F1 on NarrativeQA (0.086$\to$0.092), showing that cross-document connections benefit narrative reasoning.
\item \textbf{Community (B4$\to$IGV):} Maintains F1 while adding community structure for global queries, with minimal overhead (+0.1--0.5s).
\end{itemize}

\section{Discussion}

\textbf{When to use IGV.} IGV is most beneficial when (1) the corpus evolves over time, (2) entity overlap exists across documents, and (3) global queries require community-level reasoning. For static corpora, a single B1 build suffices.

\textbf{Limitations.} (1) The lightweight entity extractor produces noisier entities than full LLM extraction; (2) Retrieval uses keyword matching rather than graph traversal; (3) Dataset scale (300 passages) is smaller than production deployments. Future work should integrate LLM-based extraction end-to-end and evaluate on full-scale benchmarks.

\textbf{Scalability.} IGV's incremental cost is $O(m \cdot N)$ for dedup (where $m \ll N$) and $O(|S| \log |S|)$ for community repartition (where $|S| \ll N$). For a 100K-entity graph with 1K new entities, IGV processes updates in seconds vs hours for full rebuild.

\section{Conclusion}

We presented IGV, an incremental GraphRAG framework that supports efficient corpus updates through semantic deduplication, graph relinking, and incremental community repartitioning. Experiments on five multi-hop QA datasets with three random seeds demonstrate that IGV reduces entity redundancy by 13--17\% while maintaining retrieval quality within 2\% of full rebuild. Statistical significance tests confirm F1 improvements ($p<0.05$), and long-term streaming over 12 batches shows sub-9\% degradation with near-zero forgetting rate. IGV enables practical GraphRAG deployment in dynamic environments where corpora evolve continuously.
